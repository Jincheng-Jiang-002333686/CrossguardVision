## 4. Spatial reasoning  (new: `spatial_reasoning.py`)

Classifies each detected person as `crossing` / `waiting` / `no_risk` from geometry
only (these are NOT model classes). Core `classify_people(boxes, crosswalk_mask, …)`
is pure cv2/numpy (no shapely) and unit-testable; `run_image()` wraps the model;
`--selftest` runs a synthetic scene.

Design evolved over three iterations this session:
1. **Foot-region overlap + fixed-pixel dilation** — initial. Over-labeled `waiting`
   (distant far-sidewalk pedestrians swept into a fixed-pixel ring).
2. **Distance-transform + body-relative + size gate** — fixed perspective: distance
   from feet to crosswalk normalized by box height (apparent body size), and a
   min-size gate dropping people too far to be at the monitored crosswalk. Fixed an
   on-crosswalk crosser that was mislabeled `waiting` (foot 1 px off the mask), and
   removed the far-sidewalk false `waiting`s.
3. **Inner-edge crossing zone** (per user) — people also cross *between* the painted
   stripes through the intersection interior. The crossing zone = the crosswalk arms
   PLUS the interior bounded by each arm's **inner edge** (corners nearest the
   intersection centre). This deliberately does NOT use a convex hull of all crosswalk
   pixels, because that connects the arms' FAR corners with chords that cut across the
   corner curb-ramps and wrongly pull *waiters* into the crossing zone. Guard:
   `min_arms=2` (no interior from a lone crosswalk). Optional `hull_erode_frac` shrink.

Demo outputs in `runs/spatial/` (e.g. `crossguard_0001_state.jpg`) and
`runs/spatial/audit/` (8-frame set). Legend: red=crossing, orange=waiting,
green=no_risk, blue=crosswalk mask, yellow outline=crossing zone.

## 5. Adversarial visual audits (multi-agent)

Two rounds, 8 diverse frames each (unseen IMG_0232 + training frames, 1080p & 4K),
one agent per frame inspecting the annotated image + per-person data, then a synthesis.

- **Round 1 (convex-hull zone):** found a *systematic* over-extension — the hull's
  straight chords cut across corner curb-ramps, flipping ~4/8 frames' corner *waiters*
  into `crossing`. 0 missed crossings.
- **Round 2 (inner-edge zone):** **6 good / 2 acceptable / 0 bad; 0 missed crossings;
  only 1 false crossing attributable to the zone** (crossguard_0050 p3, a curb-waiter
  on a tactile strip). The other flagged case (corss2_0100 p9) is the separate
  `crossing_k` distance-fallback path, `in_zone=false`, not the zone.
- Verdict: **ship the inner-edge zone.** Optional cleanup: small `--hull-erode-frac`
  (~0.02) to remove the last curb overhang; review the `crossing_k` fallback for the
  one non-zone false crossing.
